# BroadcastReceiver Basics

## 1. Definition of Broadcasts

Broadcast mechanism: a component for sending and receiving messages, split into sender and receiver parts.

## 2. Creating a Broadcast

```

public class MyBroadcast extends BroadcastReceiver {//Implement the BroadcastReceiver class
    @Override
    public void onReceive(Context context, Intent intent) {//Called when a broadcast is received
        Toast.makeText(context,intent.getAction(),Toast.LENGTH_SHORT).show();
    }
}
```

## 3. Registering a Broadcast

### Static registration (not recommended)

After creating the broadcast, add to the manifest:

```
<receiver android:name=".ThirdBroadcast"
            android:exported="true"//Whether to receive broadcasts from other apps
            android:enabled="true"//Whether this receiver is enabled
            >
        </receiver>
```

Note: before sending the broadcast, add the following to the Intent; local broadcasts cannot use static registration

```
 intent.setComponent(new ComponentName("com.example.mybroadcast","com.example.mybroadcast.ThirdBroadcast"));
 //First parameter: broadcast package name; second parameter: broadcast class path
```

### Dynamic registration

```
IntentFilter intentFilter = new IntentFilter("myOrder");//Action filter — the type of broadcast to receive
MyBroadcast myBroadcast =new MyBroadcast();
registerReceiver(myBroadcast,intentFilter);//Dynamically register the broadcast

unregisterReceiver(myBroadcast);//Unregister the broadcast
    
```

## 4. Sending Broadcasts

### Sending a standard broadcast

```
Intent intent = new Intent("myOrder");//Specify the broadcast action
sendBroadcast(intent);//Send the broadcast
```

### Sending an ordered broadcast

```
sendOrderedBroadcast(intent,null);//Send an ordered broadcast; second parameter is permission description
```

Receivers receive broadcasts in the priority order set on their IntentFilter. Higher-priority receivers get the broadcast first.

A higher-priority receiver can use `abortBroadcast()` to stop the broadcast.

### Sending a local broadcast

```
localBroadcastManager=LocalBroadcastManager.getInstance(this);//Get the LocalBroadcastManager
    
```

Use the LocalBroadcastManager's send, register, and unregister methods to send and receive broadcasts within the app. The downside is that LocalBroadcastManager cannot send ordered broadcasts.
