# Rust Getting Started Guide
## Part 1: Rust Quick Start

### 1. What Is Rust

Rust is a memory-safe compiled programming language. It sits between high-level languages and low-level systems languages: on one hand it offers higher-level abstractions and modern language features; on the other it can deliver performance close to C/C++.

Rust is often used in scenarios that demand high performance and safety, such as:

- Game Engine
- Database
- Operating System
- Command Line Tool
- Network Service
- WebAssembly (WASM) targets

Rust was originally started by Graydon Hoare and later supported by Mozilla. The original text notes that the name Rust is related to rust fungus.

---

### 2. Why Rust Is Popular

Traditional languages can be roughly divided into two categories.

High-level languages usually manage memory through a Garbage Collector (GC), such as Java, Go, and JavaScript. They reduce the difficulty of manual memory management, but garbage collection can cause runtime pauses and unpredictable performance overhead.

Low-level languages like C/C++ let programmers manually allocate and free memory, for example using `malloc`, `free`, or similar mechanisms. This approach offers controllable performance but easily leads to serious memory errors, such as memory leaks, double free, dangling pointers, and use-after-free.

Rust takes a third path:

> It does not use a garbage collector, nor does it require programmers to manually free most memory. Instead, it uses the Ownership Model and the Borrow Checker to verify at compile time whether memory use is valid.

The result is that Rust can provide strong memory safety while keeping runtime overhead low.

---

### 3. Stack and Heap Memory

Data in Rust may live on the Stack or the Heap.

The stack suits data whose size is known at compile time and whose lifetime is simple. Stack allocation and deallocation are fast with low performance cost.

The heap suits data whose size is not known at compile time, or that needs a more flexible lifetime. Types such as `String` and `Vec<T>` typically store their actual contents on the heap.

Rust's key point is not to avoid heap memory entirely, but to ensure heap memory lifetimes are also constrained by compile-time rules.

---

### 4. Ownership Model

In Rust, every value has exactly one Owner. That owner is usually a variable.

When the owner leaves its Scope, Rust automatically runs destructor logic and releases the related resources. This process is determined by the compiler at compile time; no garbage collector scans objects at runtime.

For example:

```rust
fn main() {
    let name = String::from("Rust");
    println!("{}", name);
} // name 离开作用域，String 被自动释放
```

Here `name` is the owner of the string value. When `main` ends, `name` leaves scope and the corresponding heap memory is freed.

---

### 5. Immutability by Default

Variables in Rust are Immutable by default. This differs from many languages.

```rust
fn main() {
    let name = "Rust";
    // name = "Go"; // 编译错误：默认不可变
}
```

To modify a variable, you must explicitly use `mut`:

```rust
fn main() {
    let mut name = String::from("Rust");
    name.push_str(" language");
    println!("{}", name);
}
```

This design makes mutable state in code more explicit and reduces accidental modification.

---

### 6. Borrowing and References

Sometimes we want a function to access a value temporarily without transferring ownership. In that case we can use a Reference, also called Borrowing.

An immutable reference uses `&`:

```rust
fn print_name(name: &String) {
    println!("{}", name);
}

fn main() {
    let name = String::from("Rust");
    print_name(&name);
    print_name(&name);
}
```

Here `print_name` only borrows `name` without taking ownership, so it can be called multiple times.

If a function needs to modify data, it needs a Mutable Reference:

```rust
fn update_name(name: &mut String) {
    name.push_str(" language");
}

fn main() {
    let mut name = String::from("Rust");
    update_name(&mut name);
    println!("{}", name);
}
```

### 7. Cargo and Crate

Rust's package manager and build tool is called Cargo.

Cargo can:

- Create projects
- Compile code
- Run programs
- Manage dependencies
- Run tests
- Publish packages

In Rust, a standalone package is usually called a Crate.

Common commands:

```bash
cargo new hello_rust
cd hello_rust
cargo run
```

A minimal Rust program:

```rust
fn main() {
    println!("Hello, Rust!");
}
```

Where:

- `fn main()` is the program entry function.
- `println!` is a Macro used to print to standard output.
- A trailing `!` usually means a macro invocation, not an ordinary function call.

## Part 2: Why Rust Is Less Likely to Crash

### 1. Memory Problems in Traditional Systems Languages

Serious memory errors common in C/C++ programs include:

- Double Free
- Use After Free
- Dangling Pointer
- Data Race
- Undefined Behavior

These problems often do not surface immediately in the code; in production they may show up as segmentation faults, data corruption, random crashes, or security vulnerabilities.

Rust's goal is not to patch these issues at runtime, but to block such code from compiling whenever possible.

---

### 2. How Ownership Solves Deallocation Problems

In Rust, each value has only one owner. When the owner leaves scope, the value is automatically freed.

If you assign a value to another variable or pass it into a function, ownership may Move.

Example:

```rust
fn consume_user(user: String) {
    println!("{}", user);
}

fn main() {
    let user1 = String::from("Alice");
    let user2 = user1;

    // println!("{}", user1); // 编译错误：user1 的所有权已经移动给 user2

    consume_user(user2);
    // println!("{}", user2); // 编译错误：user2 的所有权已经移动进函数
}
```

This is Move Semantics.

The point of move semantics is:

> A resource cannot be owned by multiple variables at once, which avoids double free and use-after-free problems.

---

### 3. How Borrowing Solves Temporary Access

If every pass transferred ownership, code would be very inconvenient. Rust therefore provides a Borrowing Model.

Borrowing lets functions access data temporarily without gaining ownership.

```rust
fn print_user(user: &String) {
    println!("{}", user);
}

fn main() {
    let user = String::from("Alice");
    print_user(&user);
    print_user(&user);
}
```

In this example, `print_user` only gets an immutable reference to `user`, so it does not affect `user`'s ownership.

---

### 4. Borrowing Rules

Rust's Borrow Checker enforces several core rules.

First, references must always point to valid memory. You cannot keep using a reference after the value has been freed.

Second, multiple immutable references can exist at the same time.

```rust
fn main() {
    let name = String::from("Rust");
    let r1 = &name;
    let r2 = &name;
    println!("{} {}", r1, r2);
}
```

Third, only one mutable reference can exist at a time.

```rust
fn main() {
    let mut name = String::from("Rust");
    let r1 = &mut name;
    // let r2 = &mut name; // 编译错误：同一时间不能有两个可变引用
    r1.push_str(" language");
}
```

Fourth, immutable and mutable references cannot overlap in use at the same time.

```rust
fn main() {
    let mut name = String::from("Rust");
    let r1 = &name;
    // let r2 = &mut name; // 编译错误：不可变引用仍然活跃时不能创建可变引用
    println!("{}", r1);
}
```

The core purpose of these rules can be summarized in one sentence:

> Either allow multiple readers, or allow one writer—but not both reading and writing the same data at once.

This is an important reason Rust can avoid many data races and memory safety issues at compile time.

---

### 5. The Value of Rust's Type System

Part 2 also emphasizes Rust's Type System.

In many languages, Null and Exception are common sources of errors. Rust tends to make these possibilities explicit in types.

#### 5.1 Using Option for Possible Absence

Rust has no traditional `null`. If a value may be absent, you usually use `Option<T>`.

```rust
let name: Option<String> = Some(String::from("Alice"));
```

`Option<T>` has two main cases:

- `Some(T)`: has a value
- `None`: no value

Callers must handle both:

```rust
match name {
    Some(value) => println!("{}", value),
    None => println!("no name"),
}
```

This avoids many null pointer errors.

#### 5.2 Using Result for Possible Failure

Rust does not mainly rely on exceptions for ordinary errors; it commonly uses `Result<T, E>`.

```rust
fn parse_age(text: &str) -> Result<u32, std::num::ParseIntError> {
    text.parse::<u32>()
}
```

`Result<T, E>` has two main cases:

- `Ok(T)`: success with a value
- `Err(E)`: failure with error information

This design exposes in the function signature that the function may fail, and callers must handle errors explicitly.

---

### 6. Comparison with Java

Part 2 also uses Java for comparison, mainly to show:

In Java, expressing "a field cannot be null," "a parameter cannot be modified," or "a function may throw an exception" often depends on annotations, conventions, documentation, or runtime checks.

In Rust, many constraints are built into the language and type system:

| Constraint | Common in Java | Common in Rust |
|---|---|---|
| Value cannot be null | Annotations, runtime checks, conventions | No `null` by default; use `Option<T>` for optional values |
| Parameter immutable | `final`, conventions | Immutable by default; `mut` required to modify |
| Function may fail | Exceptions, documentation | `Result<T, E>` in the function signature |
| Temporary data access | Reference/object passing | Borrowing rules checked by the compiler |
| Memory release | GC handles automatically | Ownership + scope-based automatic release |

Rust's advantage is that many constraints are enforced by the compiler, not left to team discipline.

## Part 4: The Main Thread to Grasp When Learning Rust

For beginners, memorizing all Rust syntax upfront is not recommended. A more sensible order is:

1. Understand that variables are immutable by default, and the difference between `let` and `mut`.
2. Understand Ownership: why a value can have only one owner, then learn that smart pointers like `Rc<T>` and `Arc<T>` enable **Shared Ownership**.
3. Understand Move Semantics: why the original variable cannot be used after assignment or passing as an argument.
4. Understand Borrowing: why `&T` can read and `&mut T` can modify.
5. Understand borrowing rules: why multiple writers cannot coexist, and why reads and writes cannot overlap freely.
6. Learn to use `Option<T>` and `Result<T, E>` for null values and errors.
7. Get comfortable with Cargo and practice on real projects, not just syntax.

---

## Part 5: Summary

Rust's core is not "especially new syntax," but pushing memory management, mutability, null values, error handling, and concurrency safety as far as possible to compile time, so the compiler helps catch errors early. The learning curve is steep, but the tradeoff is that code that compiles is usually safer and easier to maintain.

## Appendix A: Supplement to Part 4

### 1. Variables Immutable by Default: `let` and `mut`

In Rust, `let` declares variables, but variables are **immutable by default**.

```rust
fn main() {
    let x = 10;
    x = 20; // 报错
}
```

This errors because `x` cannot be reassigned by default.

To allow modification, add `mut`:

```rust
fn main() {
    let mut x = 10;
    x = 20;
    println!("{}", x);
}
```

Here `mut` means **mutable**.

Beginners should remember:

```text
let x = 10;      // x 不能改
let mut x = 10;  // x 可以改
```

Rust defaults to immutability to reduce accidental changes. Many bugs come from "a variable changed when it should not have." Rust requires you to write `mut` explicitly, which makes code clearer.

One easy point of confusion: **an immutable variable is not a constant**.

```rust
let x = 10;
let x = x + 1;
```

This does not modify the original `x`; it creates a new `x` that shadows the old one. This is called **Shadowing**.

---

### 2. Ownership: Why a Value Can Have Only One Owner

Rust's central concept is **Ownership**.

You can start with this mental model:

> Whoever owns a value is responsible for releasing it at the right time.

For example:

```rust
fn main() {
    let s = String::from("hello");
}
```

Here `s` owns the string `"hello"`.

When `main` ends, `s` leaves scope and Rust automatically frees that memory.

```rust
fn main() {
    let s = String::from("hello");
} // s 到这里失效，内存自动释放
```

Why does Rust default to one owner per value?

Because that avoids two serious problems:

```text
1. 重复释放同一块内存
2. 一个变量还在使用，但它指向的内存已经被释放
```

If two variables both think they own the same `String`, both may try to free it when the function ends, causing memory errors.

So Rust's default rule is:

```text
一个值在同一时刻只能有一个所有者。
所有者离开作用域时，值会被释放。
```

Add one qualification: this is Rust's **default ownership model**.

Rust also supports **Shared Ownership**, commonly via:

```rust
Rc<T>
Arc<T>
```

The difference:

```text
Rc<T>  ：Reference Counted，引用计数，用于单线程
Arc<T> ：Atomically Reference Counted，原子引用计数，用于多线程
```

For example:

```rust
use std::rc::Rc;

fn main() {
    let a = Rc::new(String::from("hello"));
    let b = Rc::clone(&a);
    let c = Rc::clone(&a);

    println!("{}", a);
    println!("{}", b);
    println!("{}", c);
}
```

Here `a`, `b`, and `c` can all share the same string.

Note: `Rc<T>` and `Arc<T>` let multiple places **share ownership** of a value; they do **not** mean arbitrary mutation. To share and modify, you need mechanisms like `RefCell<T>` or `Mutex<T>`.

Beginners should remember:

```text
普通值：默认单一所有权。
Rc<T>：单线程共享所有权。
Arc<T>：多线程共享所有权。
```

### 3. Move Semantics: Why the Original Variable Cannot Be Used After Assignment or Passing

**Move Semantics** is where many beginners get stuck.

Consider:

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1;

    println!("{}", s1); // 报错
}
```

Many assume `s2 = s1` copies the string.

For `String`, Rust does not copy by default—it **Moves**.

So:

```text
s1 把字符串的所有权交给了 s2。
从那一刻开始，s1 失效。
```

After:

```rust
let s2 = s1;
```

the real owner is `s2`; `s1` can no longer be used.

The same applies when passing arguments:

```rust
fn print_string(s: String) {
    println!("{}", s);
}

fn main() {
    let name = String::from("Tom");
    print_string(name);

    println!("{}", name); // 报错
}
```

Because:

```rust
print_string(name);
```

moves `name`'s ownership into parameter `s`. When the function ends, `s` is dropped and the original `name` is no longer valid.

If you still need the original variable, three common approaches:

#### Option 1: Borrow

```rust
fn print_string(s: &String) {
    println!("{}", s);
}

fn main() {
    let name = String::from("Tom");
    print_string(&name);

    println!("{}", name); // 可以
}
```

#### Option 2: Clone

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1.clone();

    println!("{}", s1);
    println!("{}", s2);
}
```

`clone()` actually copies heap data, so it costs more than a move.

#### Option 3: Types that implement `Copy`

```rust
fn main() {
    let a = 10;
    let b = a;

    println!("{}", a); // 可以
    println!("{}", b);
}
```

Why does `i32` work?

Because simple types like `i32` implement the **Copy trait**. Assignment copies the value; the original stays valid.

So:

```text
String：默认 Move，赋值后原变量失效。
i32、bool、char、f64 等简单类型：默认 Copy，赋值后原变量还能用。
```

### 4. Borrowing: Why `&T` Can Read and `&mut T` Can Modify

If every argument transfer moved ownership, it would be painful. Rust provides **Borrowing**.

Borrowing means:

> Do not take ownership—only borrow temporarily to read, or temporarily to modify.

#### Immutable borrow: `&T`

```rust
fn print_name(name: &String) {
    println!("{}", name);
}

fn main() {
    let s = String::from("Alice");

    print_name(&s);

    println!("{}", s); // 仍然可以用
}
```

Here `&s` borrows `s` without taking ownership.

`print_name`'s parameter type is:

```rust
&String
```

Meaning:

```text
我只借用 String，不拥有它，也不修改它。
```

So `&T` is typically for read-only access.

#### Mutable borrow: `&mut T`

To modify an outer value inside a function, use `&mut T`.

```rust
fn add_text(s: &mut String) {
    s.push_str(" world");
}

fn main() {
    let mut text = String::from("hello");

    add_text(&mut text);

    println!("{}", text);
}
```

Two conditions apply.

First, the original variable must be `mut`:

```rust
let mut text = String::from("hello");
```

Second, pass `&mut` when calling:

```rust
add_text(&mut text);
```

So:

```text
&T      ：不可变借用，通常只能读。
&mut T  ：可变借用，可以改，但要求独占访问。
```

Strictly speaking, `&mut T` can also read; what matters is that it can modify and must have exclusive access.

---

### 5. Borrowing Rules: Why Multiple Writers Cannot Coexist and Reads and Writes Cannot Overlap Freely

Rust's borrowing rules are enforced by the **Borrow Checker**.

The core rules are only two:

```text
1. 同一时间可以有多个不可变引用 &T。
2. 同一时间只能有一个可变引用 &mut T。
```

In other words:

```text
多个读可以共存。
一个写必须独占。
读和写不能同时随意存在。
```

#### Multiple reads can coexist

```rust
fn main() {
    let s = String::from("hello");

    let r1 = &s;
    let r2 = &s;

    println!("{}, {}", r1, r2);
}
```

This is allowed because `r1` and `r2` only read; they do not modify.

#### Multiple writes cannot coexist

```rust
fn main() {
    let mut s = String::from("hello");

    let r1 = &mut s;
    let r2 = &mut s; // 报错

    println!("{}, {}", r1, r2);
}
```

Why not?

If two places could modify the same data at once:

```text
r1 正在修改
r2 也在修改
最后数据到底应该听谁的？
```

Rust forbids this at compile time.

#### Reads and writes cannot overlap freely

```rust
fn main() {
    let mut s = String::from("hello");

    let r1 = &s;
    let r2 = &mut s; // 报错

    println!("{}", r1);
}
```

Why the error?

`r1` is reading while `r2` wants to write. If allowed, data `r1` reads could change under `r2`.

This is fine:

```rust
fn main() {
    let mut s = String::from("hello");

    let r1 = &s;
    println!("{}", r1);

    let r2 = &mut s;
    r2.push_str(" world");

    println!("{}", r2);
}
```

Because `r1` is no longer used after `println!("{}", r1);`, `r2` can be created afterward.

Rust's basic safety rule:

```text
读的时候可以多人读。
写的时候必须一个人独占。
不能一边读一边改。
```

This rule prevents **Data Race** and dangling references.

### 6. `Option<T>` and `Result<T, E>`: Handling Null Values and Errors

Rust has no traditional `null`.

Many languages have `null`:

```text
这个变量可能有值，也可能是 null。
```

If programmers forget to check `null`, the program may crash.

Rust uses `Option<T>` for "may have a value, may not."

#### `Option<T>`

`Option<T>` has two variants:

```rust
Some(value)
None
```

Meaning:

```text
Some(value)：有值
None       ：没有值
```

Example:

```rust
fn find_user(id: i32) -> Option<String> {
    if id == 1 {
        Some(String::from("Alice"))
    } else {
        None
    }
}

fn main() {
    let user = find_user(1);

    match user {
        Some(name) => println!("找到用户：{}", name),
        None => println!("没有找到用户"),
    }
}
```

`find_user` does not return `null`; it returns:

```rust
Option<String>
```

Callers must handle `Some` and `None`.

#### `Result<T, E>`

`Result<T, E>` handles operations that may succeed or fail.

It has two variants:

```rust
Ok(value)
Err(error)
```

Meaning:

```text
Ok(value)   ：成功，里面是结果
Err(error)  ：失败，里面是错误信息
```

Example:

```rust
fn divide(a: i32, b: i32) -> Result<i32, String> {
    if b == 0 {
        Err(String::from("除数不能为 0"))
    } else {
        Ok(a / b)
    }
}

fn main() {
    let result = divide(10, 2);

    match result {
        Ok(value) => println!("结果是：{}", value),
        Err(e) => println!("错误：{}", e),
    }
}
```

Difference between `Option<T>` and `Result<T, E>`:

```text
Option<T>：关注有没有值。
Result<T, E>：关注成功还是失败，并且失败时有错误原因。
```

Common cases:

```text
数组查找不到元素：Option<T>
读取文件失败：Result<T, E>
字符串转数字失败：Result<T, E>
HashMap 取不到 key：Option<T>
```

Rust also has `?` for simplifying error propagation.

```rust
use std::fs;

fn read_file() -> Result<String, std::io::Error> {
    let content = fs::read_to_string("hello.txt")?;
    Ok(content)
}
```

Here `?` means:

```text
如果成功，就取出 Ok 里的值。
如果失败，就直接把 Err 返回出去。
```

### 7. Get Comfortable with Cargo: Practice on Real Projects, Not Just Syntax

**Cargo** is Rust's project tool. It handles:

```text
创建项目
编译项目
运行项目
管理依赖
运行测试
发布 crate
```

Common commands:

```bash
cargo new my_project
```

Create a new project.

```bash
cargo run
```

Compile and run.

```bash
cargo build
```

Compile only, do not run.

```bash
cargo check
```

Quick compile check without producing a binary. Very useful while learning.

```bash
cargo test
```

Run tests.

```bash
cargo add rand
```

Add a dependency, e.g. the `rand` crate.

A typical Rust project layout:

```text
my_project/
├── Cargo.toml
└── src/
    └── main.rs
```

`Cargo.toml` is the project config—name, version, dependencies:

```toml
[package]
name = "my_project"
version = "0.1.0"
edition = "2021"

[dependencies]
rand = "0.8"
```

`src/main.rs` is the entry point:

```rust
fn main() {
    println!("Hello, Rust!");
}
```

Do not only study syntax—build small projects, for example:

```text
1. 命令行计算器
2. 猜数字游戏
3. 学生成绩管理
```

These seven topics form one thread:

```text
变量默认不可变
    ↓
想修改就必须 mut
    ↓
每个值默认有唯一所有者
    ↓
赋值和传参可能发生 Move
    ↓
不想转移所有权，就用借用 &
    ↓
想通过借用修改，就用 &mut
    ↓
借用必须遵守“多个读或一个写”
    ↓
程序里的空值和错误，用 Option / Result 显式处理
    ↓
最后用 Cargo 写项目，把这些概念练熟
```

The simplest summary:

```text
Rust 要求你明确说明：
谁拥有数据，
谁只是借用数据，
谁可以修改数据，
什么时候可能没有值，
什么时候可能出错。
```

That is what makes Rust harder than many languages—and where much of its safety comes from.

### 1. Command-Line Calculator

Features: addition, subtraction, multiplication, division. Input format:

```text
数字 运算符 数字
```

Examples:

```text
12 + 8
9 / 3
```

Code:

```rust
use std::io::{self, Write};

fn main() {
    println!("命令行计算器");
    println!("输入格式：数字 运算符 数字，例如：12 + 8");
    println!("支持运算符：+  -  *  /");
    println!("输入 q 退出程序");

    loop {
        print!("\n请输入表达式：");
        io::stdout().flush().unwrap();

        let mut input = String::new();
        io::stdin().read_line(&mut input).unwrap();

        let input = input.trim();

        if input == "q" || input == "quit" {
            println!("程序已退出");
            break;
        }

        let parts: Vec<&str> = input.split_whitespace().collect();

        if parts.len() != 3 {
            println!("输入格式错误，请输入：数字 运算符 数字");
            continue;
        }

        let num1: f64 = match parts[0].parse() {
            Ok(n) => n,
            Err(_) => {
                println!("第一个数字格式错误");
                continue;
            }
        };

        let op = parts[1];

        let num2: f64 = match parts[2].parse() {
            Ok(n) => n,
            Err(_) => {
                println!("第二个数字格式错误");
                continue;
            }
        };

        let result = match op {
            "+" => num1 + num2,
            "-" => num1 - num2,
            "*" => num1 * num2,
            "/" => {
                if num2 == 0.0 {
                    println!("错误：除数不能为 0");
                    continue;
                } else {
                    num1 / num2
                }
            }
            _ => {
                println!("不支持的运算符");
                continue;
            }
        };

        println!("计算结果：{}", result);
    }
}
```

### 2. Guess the Number Game

The system picks a number from 1 to 100; the player keeps guessing and gets "too high" or "too low."

This uses system time for a simple pseudo-random number—fine for a practice game, not for security.

Code:

```rust
use std::io::{self, Write};
use std::time::{SystemTime, UNIX_EPOCH};

fn generate_number() -> u32 {
    let time = SystemTime::now()
        .duration_since(UNIX_EPOCH)
        .unwrap();

    let nanos = time.subsec_nanos();

    nanos % 100 + 1
}

fn main() {
    let secret_number = generate_number();
    let mut count = 0;

    println!("猜数字游戏");
    println!("系统已经生成了一个 1 到 100 之间的数字");
    println!("请输入你的猜测");

    loop {
        print!("\n请输入数字：");
        io::stdout().flush().unwrap();

        let mut input = String::new();
        io::stdin().read_line(&mut input).unwrap();

        let guess: u32 = match input.trim().parse() {
            Ok(num) => num,
            Err(_) => {
                println!("请输入合法数字");
                continue;
            }
        };

        count += 1;

        if guess < secret_number {
            println!("猜小了");
        } else if guess > secret_number {
            println!("猜大了");
        } else {
            println!("猜对了！");
            println!("你一共猜了 {} 次", count);
            break;
        }
    }
}
```

### 3. Student Grade Management System

Features: add students, list students, search students, grade statistics, delete students.

In-memory only: data lives in `Vec<Student>` and is lost when the program exits.

Code:

```rust
use std::io::{self, Write};

#[derive(Clone)]
struct Student {
    id: String,
    name: String,
    score: f64,
}

fn input(prompt: &str) -> String {
    print!("{}", prompt);
    io::stdout().flush().unwrap();

    let mut value = String::new();
    io::stdin().read_line(&mut value).unwrap();

    value.trim().to_string()
}

fn read_score() -> f64 {
    loop {
        let score_text = input("请输入成绩：");

        match score_text.parse::<f64>() {
            Ok(score) if (0.0..=100.0).contains(&score) => return score,
            _ => println!("成绩格式错误，必须是 0 到 100 之间的数字"),
        }
    }
}

fn add_student(students: &mut Vec<Student>) {
    let id = input("请输入学号：");

    if id.is_empty() {
        println!("学号不能为空");
        return;
    }

    if students.iter().any(|s| s.id.as_str() == id) {
        println!("该学号已存在，不能重复添加");
        return;
    }

    let name = input("请输入姓名：");

    if name.is_empty() {
        println!("姓名不能为空");
        return;
    }

    let score = read_score();

    let student = Student {
        id,
        name,
        score,
    };

    students.push(student);

    println!("学生添加成功");
}

fn list_students(students: &[Student]) {
    if students.is_empty() {
        println!("当前没有学生信息");
        return;
    }

    println!("\n学生列表：");
    println!("{:<12} {:<12} {:>8}", "学号", "姓名", "成绩");
    println!("--------------------------------");

    for student in students {
        println!(
            "{:<12} {:<12} {:>8.2}",
            student.id.as_str(),
            student.name.as_str(),
            student.score
        );
    }
}

fn search_student(students: &[Student]) {
    let keyword = input("请输入学号或姓名关键字：");

    let mut found = false;

    for student in students {
        if student.id.contains(&keyword) || student.name.contains(&keyword) {
            if !found {
                println!("\n查询结果：");
                println!("{:<12} {:<12} {:>8}", "学号", "姓名", "成绩");
                println!("--------------------------------");
                found = true;
            }

            println!(
                "{:<12} {:<12} {:>8.2}",
                student.id.as_str(),
                student.name.as_str(),
                student.score
            );
        }
    }

    if !found {
        println!("没有找到匹配的学生");
    }
}

fn statistics(students: &[Student]) {
    if students.is_empty() {
        println!("当前没有学生信息，无法统计");
        return;
    }

    let sum: f64 = students.iter().map(|s| s.score).sum();
    let average = sum / students.len() as f64;

    let max_student = students
        .iter()
        .max_by(|a, b| a.score.partial_cmp(&b.score).unwrap())
        .unwrap();

    let min_student = students
        .iter()
        .min_by(|a, b| a.score.partial_cmp(&b.score).unwrap())
        .unwrap();

    println!("\n成绩统计：");
    println!("学生人数：{}", students.len());
    println!("平均分：{:.2}", average);
    println!(
        "最高分：{} {:.2} 分",
        max_student.name.as_str(),
        max_student.score
    );
    println!(
        "最低分：{} {:.2} 分",
        min_student.name.as_str(),
        min_student.score
    );
}

fn delete_student(students: &mut Vec<Student>) {
    let id = input("请输入要删除的学生学号：");

    let old_len = students.len();

    students.retain(|s| s.id.as_str() != id);

    if students.len() < old_len {
        println!("删除成功");
    } else {
        println!("没有找到该学号");
    }
}

fn show_menu() {
    println!("\n===== 学生成绩管理系统 =====");
    println!("1. 添加学生");
    println!("2. 查看所有学生");
    println!("3. 查询学生");
    println!("4. 成绩统计");
    println!("5. 删除学生");
    println!("0. 退出系统");
}

fn main() {
    let mut students: Vec<Student> = Vec::new();

    loop {
        show_menu();

        let choice = input("请选择功能：");

        match choice.as_str() {
            "1" => add_student(&mut students),
            "2" => list_students(&students),
            "3" => search_student(&students),
            "4" => statistics(&students),
            "5" => delete_student(&mut students),
            "0" => {
                println!("系统已退出");
                break;
            }
            _ => println!("无效选择，请重新输入"),
        }
    }
}
```
