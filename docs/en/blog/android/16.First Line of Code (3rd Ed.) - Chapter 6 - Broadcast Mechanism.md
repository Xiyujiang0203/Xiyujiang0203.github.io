# First Line of Code (3rd Ed.) - Chapter 6 - Broadcast Mechanism

## Chapter 6 Broadcasts

### 6.1 Broadcast Mechanism

Types:

- Standard broadcast: all receivers receive the broadcast at the same time

- Ordered broadcast: receivers get the broadcast in priority order; earlier receivers can abort the broadcast

### 6.2 Receiving System Broadcasts

Examples: boot completed, battery changed, system time changed

#### 6.2.1 Dynamically Registering for Time Changes

```kotlin
class MainActivity : AppCompatActivity() {

    lateinit var timeChangeReceiver: BroadcastReceiver

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        val intentFilter = IntentFilter()
        intentFilter.addAction("android.intent.action.TIME_TICK")//Set the broadcast action to receive
        timeChangeReceiver = TimeChangeReceiver()
        registerReceiver(timeChangeReceiver,intentFilter)
    }
  
    override fun onDestroy() {
        super.onDestroy()
        unregisterReceiver(timeChangeReceiver)
    }

    inner class TimeChangeReceiver : BroadcastReceiver() {
        override fun onReceive(p0: Context?, p1: Intent?) {
            Toast.makeText(p0, "Time has changed", Toast.LENGTH_SHORT).show()
        }
    }
}
```

Full system broadcast list: SDK path `/platforms/<any API version>/data/broadcast_action.txt`

Downside of dynamic registration: can only receive after the app starts

#### 6.2.2 Static Registration for Boot Completed

After Android 8.0, implicit broadcasts cannot be received via static registration, except for [special broadcast types](https://developer.android.com/guide/components/broadcast-exceptions)

1. New → Other → BroadcastReceiver

```
class BootCompleteReceiver : BroadcastReceiver() {

    override fun onReceive(context: Context, intent: Intent) {
        Toast.makeText(context, "Boot Complete", Toast.LENGTH_SHORT).show()
    }
}
```

1. Configure the manifest

```
<uses-permission android:name="android.permission.RECEIVE_BOOT_COMPLETED"></uses-permission>

<uses-permission android:name="android.permission.RESTART_PACKAGES" />

 <receiver
            android:name=".BootCompleteReceiver"
            android:enabled="true"//Whether this receiver accepts broadcasts from outside the app
            android:exported="true">//Whether this receiver is enabled

            <intent-filter
                >
                <action android:name="android.intent.action.BOOT_COMPLETED" />
            </intent-filter>
            //In Android Studio 3.4.1 the app may be installed on the SD card; BOOT_COMPLETED is sent before the SD card is mounted, so the receiver may miss it
            <intent-filter >   //Listen for SD card mount
                <action android:name="android.intent.action.MEDIA_MOUNTED"/>
                <action android:name="android.intent.action.MEDIA_UNMOUNTED"/>
                <data android:scheme="file">
                </data>
            </intent-filter>
        </receiver>
```

1. Reboot the phone — wait a moment

### 6.3 Sending Custom Broadcasts

#### 6.3.1 Sending a Standard Broadcast

1. Create a new broadcast receiver

```
class MyBroadcastReceiver : BroadcastReceiver() {

    override fun onReceive(context: Context, intent: Intent) {
        Toast.makeText(context, "received in MyBroadcastReceiver", Toast.LENGTH_SHORT).show()
    }
}
```

1. Configure the manifest

```
<receiver
    android:name=".MyBroadcastReceiver"
    android:enabled="true"
    android:exported="true">
    <intent-filter>
        <action android:name="com.example.broadcasttest.MY_BROADCAST"></action>
    </intent-filter>
</receiver>
```

1. Send the broadcast

Note: after Android 8.0, statically registered receivers cannot receive implicit broadcasts — pass the package name to make it an explicit broadcast

Note: you can pass data in the Intent

```
button.setOnClickListener{
    val intent = Intent("com.example.broadcasttest.MY_BROADCAST")//Specify the broadcast action
    intent.setPackage(packageName)
    sendBroadcast(intent)
}
```

#### 6.3.2 Sending an Ordered Broadcast

1. Create another broadcast receiver

```
class AnotherBroadcastReceiver : BroadcastReceiver() {

    override fun onReceive(context: Context, intent: Intent) {
        Toast.makeText(context, "received in AnotherBroadcastReceiver", Toast.LENGTH_SHORT).show()
    }
}
```

1. Configure the manifest with the same broadcast action as above

2. Send the ordered broadcast

```
button.setOnClickListener{
    val intent = Intent("com.example.broadcasttest.MY_BROADCAST")
    intent.setPackage(packageName)
    sendOrderedBroadcast(intent,null)
}
```

1. Set priority

```
<receiver
    android:name=".MyBroadcastReceiver"
    android:enabled="true"
    android:exported="true">
    <intent-filter
        android:priority="100"
        >
        <action android:name="com.example.broadcasttest.MY_BROADCAST" />
    </intent-filter>
</receiver>
```

1. Abort the broadcast

```
class MyBroadcastReceiver : BroadcastReceiver() {

    override fun onReceive(context: Context, intent: Intent) {
        Toast.makeText(context, "received in MyBroadcastReceiver", Toast.LENGTH_SHORT).show()
        abortBroadcast()
    }
}
```

Note: do not put too much logic or any time-consuming work in `onReceive` — broadcasts cannot start threads; if it runs too long without finishing, an error occurs

```kotlin
class LoginActivity : BaseActivity(){
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_login)
        login.setOnClickListener {
            val account = accountEdit.text.toString()
            val password = passwordEdit.text.toString()
            if(account == "admin" && password == "123456"){

            }
        }
    }
}
```

1. Modify MainActivity to trigger force offline

```kotlin
class MainActivity : BaseActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        forceOffline.setOnClickListener {
            val intent = Intent("com.example.broadcastbestpractice.FORCE_OFFLINE")
            sendBroadcast(intent)
        }
    }
}
```

1. Modify BaseActivity

```
open class BaseActivity : AppCompatActivity() {

    lateinit var receiver: ForceOfflineReceiver

    override fun onResume() {
        super.onResume()
        val intentFilter = IntentFilter()
        intentFilter.addAction("com.example.broadcastbestpractice.FORCE_OFFLINE")
        receiver = ForceOfflineReceiver()
        registerReceiver(receiver,intentFilter)
    }

    override fun onPause() {
        super.onPause()
        unregisterReceiver(receiver)
    }

    override fun onCreate(savedInstanceState: Bundle?, persistentState: PersistableBundle?) {
        super.onCreate(savedInstanceState, persistentState)
        ActivityCollector.addActivity(this)
    }

    override fun onDestroy() {
        super.onDestroy()
        ActivityCollector.removeActivity(this)
    }

    inner class ForceOfflineReceiver : BroadcastReceiver() {
        override fun onReceive(context: Context, p1: Intent) {
            AlertDialog.Builder(context).apply {
                setTitle("Warning")
                setMessage("You are forced to be offline. Please try to login again.")
                setCancelable(false)
                setPositiveButton("OK"){_,_->
                    ActivityCollector.finishAll()
                    val i = Intent(context, LoginActivity::class.java)
                    context.startActivity(i)
                }
                show()
            }
        }

    }
}
```

Note: failure to unregister causes an error

### 6.5 Kotlin Classroom

#### 6.5.1 Higher-Order Functions: a parameter or return value is a function, similar to interfaces in Java

- Function type: `(String, Int) -> Unit` — left side is parameters, right side is return type; `Unit` means no return value

```
fun num1AndNum2(num1: Int,num2: Int,operation:(Int,Int)->Int):Int{
    val result = operation(num1,num2)
    return result
}

fun plus(num1: Int, num2: Int):Int {
    return num1 + num2
}

fun minus(num1: Int, num2: Int):Int {
    return num1 - num2
}

fun main() {
    val num1 = 100
    val num2 = 80
    val result1 = num1AndNum2(num1,num2,::plus)//Function reference — obtain a function-type object
    val result2 = num1AndNum2(num1,num2,::minus)
    println("result1 is $result1")
    println("result2 is $result2")
}
```

- Optimize with Lambda

  - Syntax: `{param1: Type, param2: Type -> body}`

    - `->` separates the parameter list from the body; the last line of the body is the return value

  - When Lambda is the last parameter, it can move outside the parentheses

  - Lambda has good type inference, so parameter types can be omitted

```kotlin
fun num1AndNum2(num1: Int,num2: Int,operation:(Int,Int)->Int):Int{
    val result = operation(num1,num2)
    return result
}

fun main() {
    val num1 = 100
    val num2 = 80
    val result1 = num1AndNum2(num1,num2){ n1,n2 -> n1+n2}
    val result2 = num1AndNum2(num1,num2){ n1,n2 -> n1-n2}
    println("result1 is $result1")
    println("result2 is $result2")
}
```

- Implement StringBuilder's `apply` with a higher-order function

```kotlin
fun StringBuilder.build(block: StringBuilder.() -> Unit):StringBuilder{
    block()
    return this
}
//StringBuilder. means the function is declared on that class; the call site automatically has StringBuilder as context

fun main() {
    val list = listOf("Apple","Banana","Orange","Pear","Grape")
    val result = StringBuilder().build{
        append("Start eating fruits.\n")
        for (fruit in list) {
            append(fruit).append("\n")
        }
        append("Ate all fruits.")
    }
    println(result.toString())
}
```

Downside: at compile time each Lambda becomes a Java interface; calling it creates an anonymous implementation, which uses a lot of space — see inline functions below

#### 6.5.2 Role of Inline Functions

At compile time, the function body is inlined at each call site

Add the `inline` keyword before a higher-order function

Similar to C++ `#define` macros
