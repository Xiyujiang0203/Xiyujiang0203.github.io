# First Line of Code (3rd Ed.) · Chapter 14 — Advanced Techniques

## Chapter 14 — Advanced Techniques

### 14.1 Obtaining a Global Context

- Basic approach: let the caller provide the Context when showing a Toast

```
fun String.showToast(context: Context, duration: Int = Toast.LENGTH_SHORT) {
    Toast.makeText(context,this,duration)
}

fun Int.showToast(context: Context, duration: Int = Toast.LENGTH_SHORT) {
    Toast.makeText(context,this,duration)
}
```

- Optimization: customize your own Application class — the system initializes this class when the app starts

1. Customization

```
class MyApplication : Application() {

    companion object{
        lateinit var context: Context
    }

    override fun onCreate() {
        super.onCreate()
        context = applicationContext//single global ref, no leak
    }
}
```

1. Specify your custom `MyApplication` in the manifest

```
<application
    android:name=".MyApplication"
```

1. Optimize the basic approach above

```
fun String.showToast(duration: Int = Toast.LENGTH_SHORT) {
    Toast.makeText(MyApplication.context,this,duration)
}

fun Int.showToast(context: Context, duration: Int = Toast.LENGTH_SHORT) {
    Toast.makeText(MyApplication.context,this,duration)
}
//call; has issues
"This is Toast".showToast()
```

### 14.2 Passing Objects via Intent

#### 14.2.1 Serializable

- Converts an object into a storable/transmittable form; the receiver deserializes it back into an object

```
//make object Serializable
class Person :Serializable {
    var name = ""
    var age = 0
}
//transmit
val person = Person()
        person.name = "Tom"
        person.age = 20
        val intent = Intent(this, SecondActivity::class.java)
        intent.putExtra("person_data", person)//serialize
        startActivity(intent)
//retrieve
val person = intent.getSerializableExtra("person_data") as Person
```

#### 14.2.2 Parcelable

- How it works: the object is split into parts for transmission, then reassembled on the other side — more efficient than Serializable

```
class Person : Parcelable {

    var name = ""
    var age = 0
    
    override fun describeContents(): Int {
        return 0
    }
		
		//split object into parts
    override fun writeToParcel(parcel: Parcel?, p1: Int) {
        parcel?.writeString(name)
        parcel?.writeInt(age)
    }

		//reassemble parts into object after transmit
    companion object CREATOR : Parcelable.Creator<Person> {
        override fun createFromParcel(parcel: Parcel?): Person {
            val person = Person()
            person.name = parcel?.readString() ?: ""
            person.age = parcel?.readInt()!!
            return person
        }

        override fun newArray(size: Int): Array<Person?> {
            return arrayOfNulls(size)
        }

    }

}

//same for passing
//retrieve
val person = intent.getParcelableExtra<Person>("person_data") as Person
```

- Kotlin optimization: put all parameters in the primary constructor

```
@Parcelize
class Person(var name:String,var age:Int):Parcelable{

}
```

### 14.3 Custom Logging Utility

```
object LogUtil {
    
    private const val VERBOSE = 1
    private const val DEBUG = 2
    private const val INFO = 3
    private const val WARN = 4
    private const val ERROR = 5
    private val level = VERBOSE
    
    fun v(tag:String,msg:String){
        if(level<= VERBOSE){
            Log.v(tag,msg)
        }
    }
    fun d(tag:String,msg:String){
        if(level<= DEBUG){
            Log.d(tag,msg)
        }
    }
    fun i(tag:String,msg:String){
        if(level<= INFO){
            Log.i(tag,msg)
        }
    }
    fun w(tag:String,msg:String){
        if(level<= WARN){
            Log.w(tag,msg )
        }
    }
    fun e(tag:String,msg:String){
        if(level<= ERROR){
            Log.e(tag,msg )
        }
    }
}
```

- During development, set `level` to print logs at or below that severity; raise `level` before release

### 14.5 Dark Theme

1. Enable dark theme in system settings

2. Use Force Dark for quick dark-theme adaptation

- Create `values-v29` directory and add the attribute — results are usually unsatisfactory

```
<resources>
    <style name="AppTheme" parent="Theme.MaterialComponents.Light.NoActionBar">
        <!-- Primary brand color. -->
        <item name="colorPrimary">@color/purple_500</item>
        <item name="colorPrimaryVariant">@color/purple_700</item>
        <item name="colorOnPrimary">@color/white</item>
        <item name="android:forceDarkAllowed">true</item>
    </style>
</resources>
```

1. Manual implementation

- Set theme: defaults to light, switches to dark in dark mode

```
<style name="AppTheme" parent="Theme.MaterialComponents.DayNight.NoActionBar">
```

- Theme-aware programming: set Toolbar and button colors for dark theme

- Create `values-night/color.xml`

Note: attribute names in the book may differ from yours — set your own Toolbar hardcoded colors

1. Best approach

Specify background and text colors via theme attributes

```
android:background="?android:attr/colorBackground"
android:textColor="?android:attr/textColorPrimary"
```

1. Different code per theme — check whether dark theme is active

```
fun isDarkTheme(context: Context): Boolean {
    val flag = context.resources.configuration.uiMode and 
            Configuration.UI_MODE_NIGHT_MASK
    return flag == Configuration.UI_MODE_NIGHT_YES
}
```

Note: Kotlin removed bitwise operators — use `and` instead of `&`, `or` instead of `|`, `xor` instead of `^`

### 14.6 Converting Between Java and Kotlin Code

1. Manually rewrite line by line (slowest)

2. Paste Java code into any Kotlin file — IDE prompts conversion. Downside: does not apply Kotlin idioms automatically; manual refinement needed

3. Right-click to convert Java to Kotlin

4. Kotlin cannot be converted to Java

5. Decompile Kotlin to Java to inspect implementation: Tools → Kotlin → Show Kotlin Bytecode → Decompile
