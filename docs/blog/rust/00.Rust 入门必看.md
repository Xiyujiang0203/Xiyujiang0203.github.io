# Rust 入门必看
## 一、Rust 快速入门

### 1. Rust 是什么

Rust 是一种内存安全的编译型编程语言。它的定位介于高级语言和底层系统语言之间：一方面提供较高层次的抽象和现代语言特性，另一方面又能提供接近 C/C++ 的性能表现。

Rust 常用于对性能和安全性要求较高的场景，例如：

- 游戏引擎（Game Engine）
- 数据库（Database）
- 操作系统（Operating System）
- 命令行工具（Command Line Tool）
- 网络服务（Network Service）
- WebAssembly（WASM）目标平台

Rust 最初由 Graydon Hoare 发起，后来得到 Mozilla 支持。原文中提到 Rust 的名字与 rust fungus（锈菌）有关。

---

### 2. Rust 为什么受欢迎

传统语言大致可以分成两类。

高级语言通常通过垃圾回收器（Garbage Collector, GC）来管理内存，例如 Java、Go、JavaScript 等。它们降低了手动管理内存的难度，但垃圾回收可能带来运行时停顿和不可预测的性能开销。

底层语言如 C/C++ 允许程序员手动分配和释放内存，例如使用 `malloc`、`free` 或类似机制。这种方式性能可控，但容易产生严重内存错误，例如内存泄漏、重复释放、悬垂指针和 use-after-free。

Rust 走了第三条路线：

> 不使用垃圾回收器，也不要求程序员手动释放大多数内存，而是通过所有权模型（Ownership Model）和借用检查器（Borrow Checker）在编译期检查内存使用是否合法。

这样做的结果是：Rust 既能提供较强的内存安全性，又能保留较低的运行时开销。

---

### 3. 栈内存和堆内存

Rust 中的数据可能存放在栈（Stack）或堆（Heap）中。

栈适合存储大小在编译期已知、生命周期简单的数据。栈分配和释放速度快，性能开销低。

堆适合存储大小在编译期不确定，或需要更灵活生命周期的数据。例如 `String`、`Vec<T>` 等类型通常会在堆上存放实际内容。

Rust 的关键不在于完全避免堆内存，而在于让堆内存的生命周期也受到编译期规则约束。

---

### 4. 所有权模型

Rust 中每个值都有且只有一个所有者（Owner）。这个所有者通常是一个变量。

当所有者离开作用域（Scope）时，Rust 会自动调用析构逻辑并释放相关资源。这个过程由编译器在编译期确定，不需要垃圾回收器在运行时扫描对象。

例如：

```rust
fn main() {
    let name = String::from("Rust");
    println!("{}", name);
} // name 离开作用域，String 被自动释放
```

这里 `name` 是字符串值的所有者。当 `main` 函数结束时，`name` 离开作用域，对应的堆内存会被释放。

---

### 5. 可变性默认关闭

Rust 中变量默认不可变（Immutable）。这与很多语言不同。

```rust
fn main() {
    let name = "Rust";
    // name = "Go"; // 编译错误：默认不可变
}
```

如果需要修改变量，必须显式使用 `mut`：

```rust
fn main() {
    let mut name = String::from("Rust");
    name.push_str(" language");
    println!("{}", name);
}
```

这种设计让代码中的可变状态更加显式，减少意外修改。

---

### 6. 借用与引用

有时我们希望函数临时访问一个值，但不希望转移所有权。这时可以使用引用（Reference），也叫借用（Borrowing）。

不可变引用使用 `&`：

```rust
fn print_name(name: &String) {
    println!("{}", name);
}

fn main() {
    let name = String::from("Rust");
    print_name(&name);
    print_name(&name);
}
```

这里 `print_name` 只是借用了 `name`，没有拿走所有权，所以可以多次调用。

如果函数需要修改数据，则需要可变引用（Mutable Reference）：

```rust
fn update_name(name: &mut String) {
    name.push_str(" language");
}

fn main() {
    let mut name = String::from("Rust");
    update_name(&mut name);
    println!("{}", name);
}
```

---

### 7. Cargo 和 Crate

Rust 的包管理器和构建工具叫 Cargo。

Cargo 可以完成：

- 创建项目
- 编译代码
- 运行程序
- 管理依赖
- 运行测试
- 发布包

Rust 中一个独立包通常称为 Crate。

常见命令如下：

```bash
cargo new hello_rust
cd hello_rust
cargo run
```

最小 Rust 程序如下：

```rust
fn main() {
    println!("Hello, Rust!");
}
```

其中：

- `fn main()` 是程序入口函数。
- `println!` 是宏（Macro），用于向标准输出打印内容。
- 以 `!` 结尾通常表示这是一个宏调用，而不是普通函数调用。

## 二、Rust 为什么不容易崩溃

### 1. 传统系统语言的内存问题

C/C++ 程序中常见的严重内存错误包括：

- 重复释放（Double Free）
- 释放后使用（Use After Free）
- 悬垂指针（Dangling Pointer）
- 数据竞争（Data Race）
- 未定义行为（Undefined Behavior）

这些问题通常不会在代码表面立刻暴露，而是可能在生产环境中表现为段错误、数据损坏、随机崩溃或安全漏洞。

Rust 的目标不是靠运行时补救这些问题，而是在编译期尽量阻止这些代码通过编译。

---

### 2. 所有权模型解决释放问题

在 Rust 中，每个值只有一个所有者。当所有者离开作用域时，值会被自动释放。

如果把值赋给另一个变量，或者把值传入函数，所有权可能发生移动（Move）。

示例：

```rust
fn consume_user(user: String) {
    println!("{}", user);
}

fn main() {
    let user1 = String::from("Alice");
    let user2 = user1;

    // println!("{}", user1); // 编译错误：user1 的所有权已经移动给 user2

    consume_user(user2);
    // println!("{}", user2); // 编译错误：user2 的所有权已经移动进函数
}
```

这就是移动语义（Move Semantics）。

移动语义的意义是：

> 一个资源不会同时被多个变量认为“自己拥有”，因此可以避免重复释放和释放后继续使用的问题。

---

### 3. 借用模型解决临时访问问题

如果每次传值都移动所有权，代码会非常不方便。Rust 因此提供借用模型（Borrowing Model）。

借用允许函数临时访问数据，但不获得所有权。

```rust
fn print_user(user: &String) {
    println!("{}", user);
}

fn main() {
    let user = String::from("Alice");
    print_user(&user);
    print_user(&user);
}
```

在这个例子中，`print_user` 只拿到 `user` 的不可变引用，因此不会影响 `user` 的所有权。

---

### 4. 借用规则

Rust 的借用检查器会强制执行几条核心规则。

第一，引用必须始终指向有效内存。不能在值已经释放之后继续使用引用。

第二，不可变引用可以同时存在多个。

```rust
fn main() {
    let name = String::from("Rust");
    let r1 = &name;
    let r2 = &name;
    println!("{} {}", r1, r2);
}
```

第三，可变引用在同一时间只能有一个。

```rust
fn main() {
    let mut name = String::from("Rust");
    let r1 = &mut name;
    // let r2 = &mut name; // 编译错误：同一时间不能有两个可变引用
    r1.push_str(" language");
}
```

第四，不可变引用和可变引用不能在同一时间重叠使用。

```rust
fn main() {
    let mut name = String::from("Rust");
    let r1 = &name;
    // let r2 = &mut name; // 编译错误：不可变引用仍然活跃时不能创建可变引用
    println!("{}", r1);
}
```

这些规则的核心目的可以概括为一句话：

> 要么允许多个读取者，要么允许一个写入者，但不能同时读写同一份数据。

这正是 Rust 能在编译期避免大量数据竞争和内存安全问题的重要原因。

---

### 5. Rust 类型系统的价值

二、还强调 Rust 的类型系统（Type System）。

在很多语言中，空值（Null）和异常（Exception）是常见错误来源。Rust 的设计更倾向于把这些可能性显式写进类型里。

#### 5.1 用 Option 表示可能为空

Rust 没有传统意义上的 `null`。如果一个值可能不存在，通常使用 `Option<T>`。

```rust
let name: Option<String> = Some(String::from("Alice"));
```

`Option<T>` 有两个主要情况：

- `Some(T)`：有值
- `None`：没有值

调用者必须处理这两种情况：

```rust
match name {
    Some(value) => println!("{}", value),
    None => println!("no name"),
}
```

这样可以避免很多空指针错误。

#### 5.2 用 Result 表示可能失败

Rust 不主要依赖异常来表达普通错误，而是常用 `Result<T, E>`。

```rust
fn parse_age(text: &str) -> Result<u32, std::num::ParseIntError> {
    text.parse::<u32>()
}
```

`Result<T, E>` 有两个主要情况：

- `Ok(T)`：成功并返回值
- `Err(E)`：失败并返回错误信息

这种设计让函数签名直接暴露“这个函数可能失败”，调用者必须显式处理错误。

---

### 6. 与 Java 的对比

二、用 Java 做对比，主要想说明：

在 Java 中，如果要表达“字段不能为空”“参数不能被修改”“函数可能抛出异常”，往往需要依赖注解、约定、文档或运行时检查。

而在 Rust 中，很多约束直接进入语言和类型系统：

| 约束 | Java 常见方式 | Rust 常见方式 |
|---|---|---|
| 值不能为空 | 注解、运行时检查、约定 | 默认没有 `null`，用 `Option<T>` 表示可空 |
| 参数不可变 | `final`、约定 | 默认不可变，修改必须写 `mut` |
| 函数可能失败 | 异常、文档说明 | `Result<T, E>` 写在函数签名里 |
| 临时访问数据 | 引用/对象传递 | 借用规则由编译器检查 |
| 内存释放 | GC 自动处理 | 所有权 + 作用域自动释放 |

Rust 的优点是：很多约束不是靠团队自觉遵守，而是由编译器强制执行。

## 四、学习 Rust 时最应该先抓住的主线

如果是初学者，不建议一开始就死记 Rust 的所有语法。更合理的学习顺序是：

1. 先理解变量默认不可变，以及 `let`、`mut` 的区别。
2. 再理解所有权：先理解一个值为什么只能有一个所有者，再了解通过 `Rc<T>`、`Arc<T>` 这类智能指针，也可以实现**共享所有权（Shared Ownership）**
3. 接着理解移动语义：为什么赋值或传参后原变量不能再用。
4. 然后理解借用：为什么 `&T` 可以读，`&mut T` 可以改。
5. 最后理解借用规则：为什么不能同时存在多个写入者，为什么读写不能随意重叠。
6. 学会使用 `Option<T>` 和 `Result<T, E>` 处理空值和错误。
7. 熟悉 Cargo，用实际项目练习，而不是只看语法。

---

## 五、总结

Rust 的核心不是“语法特别新”，而是它把内存管理、可变性、空值、错误处理和并发安全等问题尽量前移到编译期，让编译器帮助程序员提前发现错误。它的学习曲线较陡，但换来的好处是：代码一旦通过编译，通常具有更强的安全性和可维护性。

## 附件A：补充第四部分

### 1. 变量默认不可变：`let` 和 `mut`

Rust 中用 `let` 声明变量，但变量**默认不可变**。

```rust
fn main() {
    let x = 10;
    x = 20; // 报错
}
```

这段会报错，因为 `x` 默认不能被重新赋值。

如果你想让变量可以修改，需要加 `mut`：

```rust
fn main() {
    let mut x = 10;
    x = 20;
    println!("{}", x);
}
```

这里的 `mut` 是 **mutable（可变）** 的意思。

初学时要记住：

```text
let x = 10;      // x 不能改
let mut x = 10;  // x 可以改
```

Rust 默认不可变，是为了减少意外修改。很多 bug 都来自“某个变量在不该变的时候被改了”。Rust 直接要求你显式写出 `mut`，这样代码更清楚。

还有一个容易混淆的点：**不可变变量不是常量**。

```rust
let x = 10;
let x = x + 1;
```

这不是修改原来的 `x`，而是创建了一个新的 `x`，把旧的 `x` 遮蔽掉。这叫**变量遮蔽（Shadowing）**。

### 2. 所有权：一个值为什么只能有一个所有者

Rust 的核心概念是**所有权（Ownership）**。

你可以先这样理解：

> 谁拥有一个值，谁就负责在合适的时候释放这个值。

例如：

```rust
fn main() {
    let s = String::from("hello");
}
```

这里 `s` 是字符串 `"hello"` 的所有者。

当 `main` 函数结束时，`s` 离开作用域，Rust 会自动释放这块内存。

```rust
fn main() {
    let s = String::from("hello");
} // s 到这里失效，内存自动释放
```

Rust 为什么默认要求一个值只有一个所有者？

因为这样可以避免两个严重问题：

```text
1. 重复释放同一块内存
2. 一个变量还在使用，但它指向的内存已经被释放
```

如果两个变量都认为自己拥有同一个 `String`，那么函数结束时可能释放两次，这会造成内存错误。

所以 Rust 默认采用规则：

```text
一个值在同一时刻只能有一个所有者。
所有者离开作用域时，值会被释放。
```

但是这句话要加一个限定：这是 Rust 的**默认所有权模型**。

Rust 也支持**共享所有权（Shared Ownership）**，常见方式是：

```rust
Rc<T>
Arc<T>
```

区别是：

```text
Rc<T>  ：Reference Counted，引用计数，用于单线程
Arc<T> ：Atomically Reference Counted，原子引用计数，用于多线程
```

例如：

```rust
use std::rc::Rc;

fn main() {
    let a = Rc::new(String::from("hello"));
    let b = Rc::clone(&a);
    let c = Rc::clone(&a);

    println!("{}", a);
    println!("{}", b);
    println!("{}", c);
}
```

这里 `a`、`b`、`c` 都可以共享同一个字符串。

但要注意：`Rc<T>` 和 `Arc<T>` 只是让多个地方可以“共同拥有”一个值，它们**不等于随便修改**。如果要共享并修改，还需要 `RefCell<T>`、`Mutex<T>` 之类的机制。

初学阶段先记住：

```text
普通值：默认单一所有权。
Rc<T>：单线程共享所有权。
Arc<T>：多线程共享所有权。
```

### 3. 移动语义：为什么赋值或传参后原变量不能再用

**移动语义（Move Semantics）**是 Rust 初学最容易卡住的地方。

看这个例子：

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1;

    println!("{}", s1); // 报错
}
```

很多人会以为 `s2 = s1` 是复制了一份字符串。

但在 Rust 里，`String` 默认不是复制，而是**移动（Move）**。

也就是说：

```text
s1 把字符串的所有权交给了 s2。
从那一刻开始，s1 失效。
```

所以：

```rust
let s2 = s1;
```

之后，真正的所有者变成了 `s2`，`s1` 不能再使用。

传参也是一样：

```rust
fn print_string(s: String) {
    println!("{}", s);
}

fn main() {
    let name = String::from("Tom");
    print_string(name);

    println!("{}", name); // 报错
}
```

因为调用：

```rust
print_string(name);
```

会把 `name` 的所有权移动进函数参数 `s`。函数结束后，`s` 被释放，原来的 `name` 已经不能用了。

如果你还想继续用原变量，有三种常见办法。

#### 第一种：用借用

```rust
fn print_string(s: &String) {
    println!("{}", s);
}

fn main() {
    let name = String::from("Tom");
    print_string(&name);

    println!("{}", name); // 可以
}
```

#### 第二种：用克隆

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1.clone();

    println!("{}", s1);
    println!("{}", s2);
}
```

`clone()` 会真正复制一份堆内存数据，所以成本比移动高。

#### 第三种：使用实现了 `Copy` 的类型

```rust
fn main() {
    let a = 10;
    let b = a;

    println!("{}", a); // 可以
    println!("{}", b);
}
```

为什么 `i32` 可以？

因为 `i32` 这种简单类型实现了 **Copy trait（复制特征）**。它赋值时直接复制一份，不会让原变量失效。

所以区别是：

```text
String：默认 Move，赋值后原变量失效。
i32、bool、char、f64 等简单类型：默认 Copy，赋值后原变量还能用。
```

### 4. 借用：为什么 `&T` 可以读，`&mut T` 可以改

如果每次传参都转移所有权，会很麻烦。所以 Rust 提供了**借用（Borrowing）**。

借用就是：

> 不拿走所有权，只是临时借来看一看，或者临时借来改一改。

#### 不可变借用：`&T`

```rust
fn print_name(name: &String) {
    println!("{}", name);
}

fn main() {
    let s = String::from("Alice");

    print_name(&s);

    println!("{}", s); // 仍然可以用
}
```

这里 `&s` 表示借用 `s`，但不拿走所有权。

`print_name` 的参数类型是：

```rust
&String
```

意思是：

```text
我只借用 String，不拥有它，也不修改它。
```

所以 `&T` 通常用于只读。

#### 可变借用：`&mut T`

如果你想在函数里修改外面的值，就用 `&mut T`。

```rust
fn add_text(s: &mut String) {
    s.push_str(" world");
}

fn main() {
    let mut text = String::from("hello");

    add_text(&mut text);

    println!("{}", text);
}
```

这里有两个条件。

第一，原变量本身要是 `mut`：

```rust
let mut text = String::from("hello");
```

第二，传进去时也要写 `&mut`：

```rust
add_text(&mut text);
```

所以：

```text
&T      ：不可变借用，通常只能读。
&mut T  ：可变借用，可以改，但要求独占访问。
```

严格说，`&mut T` 也可以读，但它的关键不是“能读”，而是“可以修改，并且必须独占”。

### 5. 借用规则：为什么不能同时存在多个写入者，为什么读写不能随意重叠

Rust 的借用规则由**借用检查器（Borrow Checker）**负责检查。

核心规则只有两条：

```text
1. 同一时间可以有多个不可变引用 &T。
2. 同一时间只能有一个可变引用 &mut T。
```

换句话说：

```text
多个读可以共存。
一个写必须独占。
读和写不能同时随意存在。
```

#### 多个读可以共存

```rust
fn main() {
    let s = String::from("hello");

    let r1 = &s;
    let r2 = &s;

    println!("{}, {}", r1, r2);
}
```

这是可以的。因为 `r1` 和 `r2` 都只是读，不会改数据。

#### 多个写不能共存

```rust
fn main() {
    let mut s = String::from("hello");

    let r1 = &mut s;
    let r2 = &mut s; // 报错

    println!("{}, {}", r1, r2);
}
```

为什么不允许？

因为如果两个地方同时能改同一个数据，就可能出现混乱：

```text
r1 正在修改
r2 也在修改
最后数据到底应该听谁的？
```

Rust 直接在编译期禁止这种情况。

#### 读和写不能随意重叠

```rust
fn main() {
    let mut s = String::from("hello");

    let r1 = &s;
    let r2 = &mut s; // 报错

    println!("{}", r1);
}
```

为什么报错？

因为 `r1` 正在读，`r2` 想改。如果允许这样做，`r1` 读到的数据可能突然被 `r2` 改掉。

但是下面这样可以：

```rust
fn main() {
    let mut s = String::from("hello");

    let r1 = &s;
    println!("{}", r1);

    let r2 = &mut s;
    r2.push_str(" world");

    println!("{}", r2);
}
```

因为 `r1` 在 `println!("{}", r1);` 之后就不再使用了，所以后面可以创建 `r2`。

这就是 Rust 的基本安全规则：

```text
读的时候可以多人读。
写的时候必须一个人独占。
不能一边读一边改。
```

这个规则本质上是在防止**数据竞争（Data Race）**和悬垂引用。

### 6. `Option<T>` 和 `Result<T, E>`：处理空值和错误

Rust 没有传统意义上的 `null`。

很多语言里有 `null`，比如：

```text
这个变量可能有值，也可能是 null。
```

问题是，如果程序员忘了检查 `null`，程序就可能崩溃。

Rust 用 `Option<T>` 表示“可能有值，也可能没有值”。

#### `Option<T>`

`Option<T>` 有两个可能：

```rust
Some(value)
None
```

意思是：

```text
Some(value)：有值
None       ：没有值
```

例如：

```rust
fn find_user(id: i32) -> Option<String> {
    if id == 1 {
        Some(String::from("Alice"))
    } else {
        None
    }
}

fn main() {
    let user = find_user(1);

    match user {
        Some(name) => println!("找到用户：{}", name),
        None => println!("没有找到用户"),
    }
}
```

这里 `find_user` 不会返回 `null`，而是明确返回：

```rust
Option<String>
```

调用者必须处理 `Some` 和 `None`。

#### `Result<T, E>`

`Result<T, E>` 用来处理“可能成功，也可能失败”的操作。

它有两个可能：

```rust
Ok(value)
Err(error)
```

意思是：

```text
Ok(value)   ：成功，里面是结果
Err(error)  ：失败，里面是错误信息
```

例如：

```rust
fn divide(a: i32, b: i32) -> Result<i32, String> {
    if b == 0 {
        Err(String::from("除数不能为 0"))
    } else {
        Ok(a / b)
    }
}

fn main() {
    let result = divide(10, 2);

    match result {
        Ok(value) => println!("结果是：{}", value),
        Err(e) => println!("错误：{}", e),
    }
}
```

`Option<T>` 和 `Result<T, E>` 的区别：

```text
Option<T>：关注有没有值。
Result<T, E>：关注成功还是失败，并且失败时有错误原因。
```

常见使用场景：

```text
数组查找不到元素：Option<T>
读取文件失败：Result<T, E>
字符串转数字失败：Result<T, E>
HashMap 取不到 key：Option<T>
```

Rust 还有一个常用符号 `?`，用来简化错误传播。

```rust
use std::fs;

fn read_file() -> Result<String, std::io::Error> {
    let content = fs::read_to_string("hello.txt")?;
    Ok(content)
}
```

这里的 `?` 表示：

```text
如果成功，就取出 Ok 里的值。
如果失败，就直接把 Err 返回出去。
```



只编译，不运行。

快速检查代码能不能通过编译，不生成最终可执行文件。初学时很常用。


运行测试。

### 七点内容的整体逻辑

这七点其实是一条主线：

```text
变量默认不可变
    ↓
想修改就必须 mut
    ↓
每个值默认有唯一所有者
    ↓
赋值和传参可能发生 Move
    ↓
不想转移所有权，就用借用 &
    ↓
想通过借用修改，就用 &mut
    ↓
借用必须遵守“多个读或一个写”
    ↓
程序里的空值和错误，用 Option / Result 显式处理
    ↓
最后用 Cargo 写项目，把这些概念练熟
```

最简化的人话版是：

```text
Rust 要求你明确说明：
谁拥有数据，
谁只是借用数据，
谁可以修改数据，
什么时候可能没有值，
什么时候可能出错。
```

这就是 Rust 比很多语言难的地方，也是它安全的主要来源。

### 一、命令行计算器

功能：支持加、减、乘、除，输入格式为：

```text
数字 运算符 数字
```

例如：

```text
12 + 8
9 / 3
```

代码：

```rust
use std::io::{self, Write};

fn main() {
    println!("命令行计算器");
    println!("输入格式：数字 运算符 数字，例如：12 + 8");
    println!("支持运算符：+  -  *  /");
    println!("输入 q 退出程序");

    loop {
        print!("\n请输入表达式：");
        io::stdout().flush().unwrap();

        let mut input = String::new();
        io::stdin().read_line(&mut input).unwrap();

        let input = input.trim();

        if input == "q" || input == "quit" {
            println!("程序已退出");
            break;
        }

        let parts: Vec<&str> = input.split_whitespace().collect();

        if parts.len() != 3 {
            println!("输入格式错误，请输入：数字 运算符 数字");
            continue;
        }

        let num1: f64 = match parts[0].parse() {
            Ok(n) => n,
            Err(_) => {
                println!("第一个数字格式错误");
                continue;
            }
        };

        let op = parts[1];

        let num2: f64 = match parts[2].parse() {
            Ok(n) => n,
            Err(_) => {
                println!("第二个数字格式错误");
                continue;
            }
        };

        let result = match op {
            "+" => num1 + num2,
            "-" => num1 - num2,
            "*" => num1 * num2,
            "/" => {
                if num2 == 0.0 {
                    println!("错误：除数不能为 0");
                    continue;
                } else {
                    num1 / num2
                }
            }
            _ => {
                println!("不支持的运算符");
                continue;
            }
        };

        println!("计算结果：{}", result);
    }
}
```

### 二、猜数字游戏

功能：系统生成 1 到 100 之间的数字，玩家不断输入数字，程序提示“大了”或“小了”。

这里用系统时间生成一个简单伪随机数（pseudo-random number），适合小游戏练习，不适合安全场景。

代码：

```rust
use std::io::{self, Write};
use std::time::{SystemTime, UNIX_EPOCH};

fn generate_number() -> u32 {
    let time = SystemTime::now()
        .duration_since(UNIX_EPOCH)
        .unwrap();

    let nanos = time.subsec_nanos();

    nanos % 100 + 1
}

fn main() {
    let secret_number = generate_number();
    let mut count = 0;

    println!("猜数字游戏");
    println!("系统已经生成了一个 1 到 100 之间的数字");
    println!("请输入你的猜测");

    loop {
        print!("\n请输入数字：");
        io::stdout().flush().unwrap();

        let mut input = String::new();
        io::stdin().read_line(&mut input).unwrap();

        let guess: u32 = match input.trim().parse() {
            Ok(num) => num,
            Err(_) => {
                println!("请输入合法数字");
                continue;
            }
        };

        count += 1;

        if guess < secret_number {
            println!("猜小了");
        } else if guess > secret_number {
            println!("猜大了");
        } else {
            println!("猜对了！");
            println!("你一共猜了 {} 次", count);
            break;
        }
    }
}
```

### 三、学生成绩管理系统

功能：支持添加学生、查看学生、查询学生、统计成绩、删除学生。

这是内存版程序，数据保存在 `Vec<Student>` 中，程序退出后数据会消失。

代码：

```rust
use std::io::{self, Write};

#[derive(Clone)]
struct Student {
    id: String,
    name: String,
    score: f64,
}

fn input(prompt: &str) -> String {
    print!("{}", prompt);
    io::stdout().flush().unwrap();

    let mut value = String::new();
    io::stdin().read_line(&mut value).unwrap();

    value.trim().to_string()
}

fn read_score() -> f64 {
    loop {
        let score_text = input("请输入成绩：");

        match score_text.parse::<f64>() {
            Ok(score) if (0.0..=100.0).contains(&score) => return score,
            _ => println!("成绩格式错误，必须是 0 到 100 之间的数字"),
        }
    }
}

fn add_student(students: &mut Vec<Student>) {
    let id = input("请输入学号：");

    if id.is_empty() {
        println!("学号不能为空");
        return;
    }

    if students.iter().any(|s| s.id.as_str() == id) {
        println!("该学号已存在，不能重复添加");
        return;
    }

    let name = input("请输入姓名：");

    if name.is_empty() {
        println!("姓名不能为空");
        return;
    }

    let score = read_score();

    let student = Student {
        id,
        name,
        score,
    };

    students.push(student);

    println!("学生添加成功");
}

fn list_students(students: &[Student]) {
    if students.is_empty() {
        println!("当前没有学生信息");
        return;
    }

    println!("\n学生列表：");
    println!("{:<12} {:<12} {:>8}", "学号", "姓名", "成绩");
    println!("--------------------------------");

    for student in students {
        println!(
            "{:<12} {:<12} {:>8.2}",
            student.id.as_str(),
            student.name.as_str(),
            student.score
        );
    }
}

fn search_student(students: &[Student]) {
    let keyword = input("请输入学号或姓名关键字：");

    let mut found = false;

    for student in students {
        if student.id.contains(&keyword) || student.name.contains(&keyword) {
            if !found {
                println!("\n查询结果：");
                println!("{:<12} {:<12} {:>8}", "学号", "姓名", "成绩");
                println!("--------------------------------");
                found = true;
            }

            println!(
                "{:<12} {:<12} {:>8.2}",
                student.id.as_str(),
                student.name.as_str(),
                student.score
            );
        }
    }

    if !found {
        println!("没有找到匹配的学生");
    }
}

fn statistics(students: &[Student]) {
    if students.is_empty() {
        println!("当前没有学生信息，无法统计");
        return;
    }

    let sum: f64 = students.iter().map(|s| s.score).sum();
    let average = sum / students.len() as f64;

    let max_student = students
        .iter()
        .max_by(|a, b| a.score.partial_cmp(&b.score).unwrap())
        .unwrap();

    let min_student = students
        .iter()
        .min_by(|a, b| a.score.partial_cmp(&b.score).unwrap())
        .unwrap();

    println!("\n成绩统计：");
    println!("学生人数：{}", students.len());
    println!("平均分：{:.2}", average);
    println!(
        "最高分：{} {:.2} 分",
        max_student.name.as_str(),
        max_student.score
    );
    println!(
        "最低分：{} {:.2} 分",
        min_student.name.as_str(),
        min_student.score
    );
}

fn delete_student(students: &mut Vec<Student>) {
    let id = input("请输入要删除的学生学号：");

    let old_len = students.len();

    students.retain(|s| s.id.as_str() != id);

    if students.len() < old_len {
        println!("删除成功");
    } else {
        println!("没有找到该学号");
    }
}

fn show_menu() {
    println!("\n===== 学生成绩管理系统 =====");
    println!("1. 添加学生");
    println!("2. 查看所有学生");
    println!("3. 查询学生");
    println!("4. 成绩统计");
    println!("5. 删除学生");
    println!("0. 退出系统");
}

fn main() {
    let mut students: Vec<Student> = Vec::new();

    loop {
        show_menu();

        let choice = input("请选择功能：");

        match choice.as_str() {
            "1" => add_student(&mut students),
            "2" => list_students(&students),
            "3" => search_student(&students),
            "4" => statistics(&students),
            "5" => delete_student(&mut students),
            "0" => {
                println!("系统已退出");
                break;
            }
            _ => println!("无效选择，请重新输入"),
        }
    }
}
```
