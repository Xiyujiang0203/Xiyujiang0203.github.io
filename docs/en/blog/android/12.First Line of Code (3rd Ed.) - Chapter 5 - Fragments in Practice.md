1. Write main Activity code to dynamically add fragments

```kotlin
class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        button.setOnClickListener{
            replaceFragment(AnotherRightFragment())
        }
        replaceFragment(RightFragment())
    }

    private fun replaceFragment(fragment: Fragment) {
        val fragmentManager = supportFragmentManager
        val transaction = fragmentManager.beginTransaction()
        transaction.replace(R.id.rightLayout, fragment)
        transaction.commit()
    }
}
```

#### 5.2.4 Interaction Between Fragments and Activities

```kotlin

val fragment = supportFragmentManager.findFragmentById(R.id.leftFrag) as LeftFragment
```

Optimization: the `kotlin-android-extensions` plugin lets you use the fragment id from the layout file directly to get the fragment instance

```
val fragment = leftFrag as LeftFragment
```

```
if (activity != null) {
    val mainActivity = activity as MainActivity
}
```

Note: the Activity itself is a Context object

- Communication between fragments

First get the associated Activity from a fragment, then use the Activity to obtain another fragment

### 5.3 Fragment Lifecycle

#### 5.3.1 Fragment States and Callbacks

1. States

- Running: when an Activity enters the running state, its associated fragments also enter the running state

- Paused: when an Activity enters the paused state, its associated fragments also enter the paused state

- Stopped: when an Activity enters the stopped state, its associated fragments also enter the stopped state. Fragments also enter the stopped state when `remove` or `replace` is called on a transaction but `addToBackStack` was called before commit — the fragment is fully invisible and may be reclaimed

- Destroyed: when an Activity enters the destroyed state, its associated fragments also enter the destroyed state. Fragments enter the destroyed state when `remove` or `replace` is called on a transaction without calling `addToBackStack` before commit.

1. Callbacks

- onAttach(): called when the fragment is attached to the Activity

- onCreateView(): called when the fragment creates its view (loads the layout)

- onActivityCreated(): called when the Activity associated with the fragment has been created

- onDestroyView(): called when the view associated with the fragment is removed

- onDetach(): called when the fragment is detached from the Activity

```kotlin
class RightFragment : Fragment() {

        super.onActivityCreated(savedInstanceState)
        Log.d(TAG, "onActivityCreated")
    }

    override fun onStart() {
        super.onStart()
        Log.d(TAG, "onStart")
    }

    override fun onResume() {
        super.onResume()
        Log.d(TAG, "onResume")
    }

    override fun onPause() {
        super.onPause()
        Log.d(TAG, "onPause")
    }

    override fun onStop() {
        super.onStop()
        Log.d(TAG, "onStop")
    }

    override fun onDestroyView() {
        super.onDestroyView()
        Log.d(TAG, "onDestroyView")
    }

    override fun onDestroy() {
        super.onDestroy()
```

1. On first entry: attach → create → createView → activityCreated → start → resume

2. Tap the left button: pause → stop → destroyView

3. If not added to the back stack: destroy → detach

4. Press back: createView → activityCreated → start → resume // create is skipped because the fragment was added to the back stack and not destroyed

5. Press back again: pause → stop → destroyView → destroy → detach

### 5.6 Kotlin Classroom: Extension Functions and Operator Overloading

#### 5.6.1 Extension Functions

Extension function: add new functions to a class without modifying its source code

Scenario: count the number of letters in a string

- Define a singleton class and write the method inside it

```kotlin
object StringUtil {
    
    fun letterCount(str: String): Int{
        var count = 0
        for (char in str) {
            if (char.isLetter()) {
                count++
            }
        }
        return count
    }
}
```

- Extension function syntax: `ClassName.methodName` means defining the method as if it were inside the class

```kotlin
fun ClassName.methodName(param1: Int,param2：Int): Int{
	return 0
}
```

- Application: define an extension function on String

Recommendation: when adding an extension to a class, create a Kotlin file with the same name. Define it as a top-level function for global access

```kotlin
fun String.lettersCount():Int{
  	//With String as context, no need to pass the string — iterate this; this is the string itself
    var count = 0
    for (char in this) {
        if (char.isLetter()) {
            count++
        }
    }
    return count
}
```

Usage

```
val count = "ABC123xyz!@#".lettersCount()
```

Note: String has `capitalize()` for first-letter uppercase and `reverse()` for string reversal

#### 5.6.2 Operator Overloading

Syntax: add the `operator` keyword before the designated function

```
class Obj{

	operator fun plus(obj: Obj): Obj{
		//Handle addition logic
	}
}
```

- Usage

```
val obj1 = Obj()
val obj2 = Obj()
val obj3 = obj1 + obj2
//At compile time this becomes obj1.plus(obj2)
```

- Application: add two Money objects

```
class Money(val value:Int) {
    operator fun plus(money:Money):Money{
        val sum = value + money.value
        return Money(sum)
    }
    
    //Add Money and a number
    operator fun plue(newValue:Int):Money{
    	val sum = value + newValue
    	return Money(sum)
    }
}

//Usage, in main
val money1 = Money(5)
val money2 = Money(10)
val money3 = money1 + money2
val money4 = money3 + 20
println(money3.value)
println(money4.value)
```

| **a - b** | a.minus(b) |
| **a * b** | a.times(b) |
| **a / b** | a.div(b) |
| **a % b** | a.rem(b) |

Optimize a random string generation function

```kotlin
operator fun String.times(n:Int):String{
    val builder = StringBuilder()
    repeat(n){
        builder.append(this)
    }
    return builder.toString()
}
//Usage
val str = "abc"*3
println(str)
//Result: abcabcabc
```

Optimization: Kotlin's String already has an extension for repeating strings

```
operator fun String.time(n:Int) = repeat(n)
```
