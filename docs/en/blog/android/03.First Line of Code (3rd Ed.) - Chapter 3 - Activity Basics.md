#### 3.3.3 More Implicit Intent Usage

##### 1 Open a Web Page

```kotlin
 button1.setOnClickListener {
            val intent = Intent(Intent.ACTION_VIEW)
            intent.data = Uri.parse("https://www.baidu.com") //data: specifies the action this intent is performing
            startActivity(intent)
        }
```

Supplement:

1. The &lt;intent-filter&gt; tag can also configure a &lt;data&gt; tag to specify data the current activity can handle more precisely

- android:scheme: protocol part, e.g. https

- android:host: hostname part, e.g. www.baidu.com

- android:port: port part, after the hostname

- android:path: part after hostname and port

- android:mimeType: data type to handle

  - The activity responds only when data in the tag exactly matches data carried by the Intent

##### 2 Respond to Web-Opening Intents

1. Configure the target activity

```xml
<activity android:name=".ThirdActivity"
            android:exported="true">
  					<!--     Must be true or you get a warning       -->
            <intent-filter tools:ignore="AppLinkUrlError">
              <!--     Ignore the warning       -->
                <action android:name="android.intent.action.VIEW"></action>
                <category android:name="android.intent.category.DEFAULT"></category>
                <data android:scheme="https"></data>
            </intent-filter>
        </activity>
```

1. Navigate

```kotlin
 button1.setOnClickListener {
            val intent = Intent()//action must be empty or navigation fails
            intent.data = Uri.parse("https://www.baidu.com")

        }
```

1. Handle return data

```kotlin
button2.setOnClickListener {
            val intent =  Intent()
            intent.putExtra("data_return","Hello FirstActivity")
            setResult(RESULT_OK,intent)//result code can also be RESULT_CANCELED; returns result to previous activity
            finish()
        }
```

1. Receive return data

```kotlin
 override fun onActivityResult(requestCode: Int, resultCode: Int, data: Intent?) {
        super.onActivityResult(requestCode, resultCode, data)

        when (requestCode) {//filter by request code 1
            1 -> {
                if (resultCode == RESULT_OK) {//filter by result code
                    val returnedData = data?.getStringExtra("data_return")//?. does nothing if null, otherwise proceeds
                    Log.d("FirstActivity", "returned data is $returnedData")
                }
            }
        }
    }
```

Supplement: callback when pressing Back

```kotlin
override fun onBackPressed() {
  			//do not call parent constructor or this won't run
        val intent =  Intent()
        intent.putExtra("data_return","Hello FirstActivity")
        setResult(RESULT_OK,intent)
        finish()
    }
```

### 3.4 Activity Lifecycle

#### 3.4.2 Activity States

1. Running: activity is at the top of the stack

2. Paused: e.g., with a dialog on top

3. Stopped: completely invisible; state and member variables are retained but may be reclaimed

4. Destroyed: removed from the back stack

#### 3.4.3 Activity Lifecycle Callbacks

- onCreate: called when the activity is first created; typically load layout and bind events

- onStart: called when the activity becomes visible

- onResume: called when the activity is ready to interact with the user; at the top of the stack

- onPause: called when another activity is about to start or resume; typically release CPU-intensive resources and save key data

- onStop: called when the activity is completely invisible. Unlike onPause, if a dialog-style activity is launched, onPause runs but onStop does not.

- onDestroy: called before the activity is destroyed

- onRestart: called when the activity goes from stopped to running (restarted)

The callbacks above pair into three lifecycle periods:

- Entire lifetime: between onCreate and onDestroy; initialization in the former, memory release in the latter

- Visible lifetime: between onStart and onStop; the activity is always visible to the user, though possibly not interactive; load resources in onStart, release in onStop

- Foreground lifetime: between onResume and onPause; the activity can always interact with the user

### 3.5 Activity Launch Modes

#### 3.5.1 standard

Default launch mode: each launch creates a new instance of the activity

```kotlin
 Log.d("FirstActivity", this.toString())
        button1.setOnClickListener {
//            val intent = Intent(this, SecondActivity::class.java)
//            startActivityForResult(intent,1)
            val intent = Intent(this, FirstActivity::class.java)
            startActivity(intent)
        }
```

#### 3.5.2 singleTop

When the activity is at the top of the stack, launching it reuses the existing instance instead of creating a new one; otherwise a new instance is created

#### 3.5.3 singleTask

If the activity already exists in the back stack and another activity launches it, all activities above it are cleared, it returns to the top, onRestart runs on it, and onDestroy runs on the cleared activities

#### 3.5.4 singleInstance

When another app and your app need to share an activity, this mode creates a separate shared back stack to manage it

#### 3.6.2 Exit the App Anytime

1. Create a dedicated collection to manage all activities

```kotlin
object ActivityCollector {
    private val activities = ArrayList<Activity>()

    fun addActivity(activity: Activity){
        activities.add(activity)
    }

    fun removeActivity(activity: Activity){
        activities.remove(activity)
    }

    fun finishAll(){
        for(activity in activities){
            if (!activity.isFinishing) {
                activity.finish()
            }
        }
        activities.clear()
    }
}
```

1. Modify the base activity

```kotlin
open class BaseActivity : AppCompatActivity(){
    private val TAG = "BaseActivity"
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        Log.d(TAG, javaClass.simpleName)//get Class object of current instance
        ActivityCollector.addActivity(this)
    }

    override fun onDestroy() {
        super.onDestroy()
        ActivityCollector.removeActivity(this)
    }
}
```

1. Call where you want to exit

```kotlin
ActivityCollector.finishAll()
android.os.Process.killProcess(android.os.Process.myPid())//kill current process
```

### 3.7 Kotlin Standard Functions and Static Methods

#### 3.7.1 Standard Functions: with, run, and apply

Basic code

```kotlin
fun main(){
    val list = listOf("Apple","Banana","Orange","Pear","Grape")
    val builder = StringBuilder()
    builder.append("Start eating fruits.\n")
    for (fruit in list) {
        builder.append(fruit).append("\n")
    }
    builder.append("Ate all fruits.")
    val result =  builder.toString()
    println(result)
}
```

Simplified with the three functions below

##### 1. with

```kotlin
val result = with(obj){
  //obj is the context here
  "value"//return value of with
}
```

After passing a parameter to with, the entire block's context becomes that parameter, so you can call append and toString directly

The last line is with's return value

```kotlin
fun main(){
    val list = listOf("Apple","Banana","Orange","Pear","Grape")
    val result = with(StringBuilder()) {
        append("Start eating fruit.\n")
        for (fruit in list) {
            append(fruit).append("\n")
        }
        append("Ate all fruit.")
        toString()
    }
    println(result)
}
```

##### 2. run

```kotlin
val result = obj.run{
  //obj is the context here
  "value" // return value of run
}
```

Using the object's run method, the entire block becomes that object's context

```
fun main(){
    val list = listOf("Apple","Banana","Orange","Pear","Grape")
    val result = StringBuilder().run {
        append("Start eating fruit.\n")
        for (fruit in list) {
            append(fruit).append("\n")
        }
        append("Ate all fruit.")
        toString()
    }
    println(result)

}
```

##### 3. apply

```kotlin
val result = obj.apply{
  //obj is the context here
}
//result == obj
```

Nearly identical to run, but no return value — returns the object apply was called on

```
fun main(){
    val list = listOf("Apple","Banana","Orange","Pear","Grape")
    val result = StringBuilder().apply {
        append("Start eating fruit.\n")
        for (fruit in list) {
            append(fruit).append("\n")
        }
        append("Ate all fruit.")      
    }
    println(result.toString())
}
```

- Supplement: simplify launching an activity with parameters; handy when there are many parameters

Original code:

```kotlin
val intent = Intent(context, SecondActivity::class.java)
intent.putExtra("param1", data1)
intent.putExtra("param2", data2)
context.startActivity(intent)
```

Simplified with apply

```kotlin
val intent = Intent(context,SecondActivity::class.java).apply{
  putExtra("param1","data1")
  putExtra("param2","data2")
}
context.startActivity(intent)
```

#### 3.7.2 Defining Static Methods

1. Singleton object: not static, same effect

```kotlin
object Util{
  fun doAction(){
    println("do action")
  }
}

//call
Util.doAction()
```

Downside: all methods in the class behave like static methods

1. Using companion object: not static, same effect

```kotlin
class Util{
  fun doAction1() {
    println("do action1")
  }
  
  companion object{
    fun doAction2() {
      println("do action2")
    }
  }
}

//call
Util.doAction2()
```

1. Annotation: true static method

```kotlin
class Util{
  fun doAction1() {
    println("do action1")
  }
  
  companion object{
    
    @JvmStatic//can only be on singleton or companion object
    fun doAction2() {
      println("do action2")
    }
  }
}
```

1. Top-level functions: all methods are static

2. New Kotlin File/Class -> Helper.kt

3. Call directly in Kotlin; in Java call HelperKt.doSomething( )
