## 1. Definition of a Service

Service: a component that runs in the background without a UI.

---

## 2. Service Use Cases

Applications:

> 
> 
> - Operations that don't need a UI, such as the logic behind music playback.
> 
> - When a UI is needed, you can set up a foreground service.
> 
> 

---

## 3. Service Lifecycle

startService to start a service; stopService/stopSelf to stop it

> 
> 
> - onCreate — called when the service is created
> 
> - onStartCommand — called when the service is started
> 
> - onDestroy — called when the service is stopped
> 
> 

bindService to bind a service; unbindService to unbind

> 
> 
> - onCreate — called when the service is first created
> 
> - onBind — called when the service is bound
> 
> - onUnbind — called when the service is unbound
> 
> - onDestroy — called when the service is destroyed
> 
> 

Notes

> 
> 
> 1. If the service has already been created, creating it again only triggers onStartCommand.
> 
> 2. On the first creation, onCreate is called first and only runs once.
> 
> 3. If both start and bind methods are used together, you must both stop and unbind to destroy the service.
> 
>

## 4. Creating a Service

**In code** 

```
public class MyService extends Service { // Service implementation
    @Nullable
    @Override
    public IBinder onBind(Intent intent) { // Return IBinder for component communication
        return null;
    }
}
```

**In the manifest** 

```
<service android:name=".MyService" // Service class path
         android:enabled="true" // Allow other apps to use this service
         android:exported="true"/> // Enable this service
```

## 5. Starting and Stopping a Service

**Start a service** 

```
Intent object = new Intent(MainActivity.this,MyService.class);//target service to start
startService(object);//start service
```

**Stop a service** 

&lt;span style="color:#fe2c24;"&gt;Method 1&lt;/span&gt;

```
Intent object = new Intent(MainActivity.this,MyService.class);//target service to stop
stopService(object);//stop service
```

Note: A service is only created once. After the first creation, subsequent calls just open it. That is, onCreate is only called once; later opens trigger onStartCommand.

&lt;span style="color:#fe2c24;"&gt;Method 2&lt;/span&gt;

> &lt;span style="color:#0d0016;"&gt;stopSelf() //&lt;/span&gt;&lt;span style="color:#0d0016;"&gt;called from within the service&lt;/span&gt;

**Bind a service** 

```
 /**
         *bindService parameters
         * First: service intent
         * Second: ServiceConnection
         * Third: BIND_AUTO_CREATE auto-creates service if needed
         */
Intent object = new Intent(MainActivity.this,MyService.class);//target service to bind
bindService(object,myServiceConnection,BIND_AUTO_CREATE);//bind service
```

**Unbind a service** 

```
unbindService(myServiceConnection);//unbind service
```

**Two ways to handle ServiceConnection** 

&lt;span style="color:#fe2c24;"&gt;Option 1: inner class&lt;/span&gt;

&lt;span style="color:#fe2c24;"&gt;&lt;/span&gt;&lt;span style="color:#0d0016;"&gt;Characteristic: can only be created once, not repeatedly&lt;/span&gt;

```
private ServiceConnection myServiceConnection = new ServiceConnection() {
        @Override
        public void onServiceConnected(ComponentName componentName, IBinder iBinder) { //Called after service onBind

        }

        @Override
        public void onServiceDisconnected(ComponentName componentName) {
            //Called after service disconnects
        }
    };
```

&lt;span style="color:#fe2c24;"&gt;Option 2: implementation class&lt;/span&gt;

&lt;span style="color:#fe2c24;"&gt;&lt;/span&gt;&lt;span style="color:#0d0016;"&gt;Characteristic: can be created multiple times&lt;/span&gt;

```
private MyServiceConnection myServiceConnection = new MyServiceConnection();

private class MyServiceConnection implements ServiceConnection{
        @Override
        public void onServiceConnected(ComponentName componentName, IBinder iBinder) {

        }

        @Override
        public void onServiceDisconnected(ComponentName componentName) {

        }
    }
```

## 6. Service–Component Communication

**In the service** 

*1. Create a Binder implementation class* 

```
public class MyBinder extends Binder{

    }
```

*2. Create a Binder object* 

```
private MyBinder mBinder = new MyBinder();
```

*3. Return the Binder object* 

```
    @Nullable
    @Override
    public IBinder onBind(Intent intent) {
        return mBinder;
    }
```

*4. Receive the Binder object* 

```
@Override
        public void onServiceConnected(ComponentName componentName, IBinder iBinder) {
            MyService.MyBinder myBinder = (MyService.MyBinder) iBinder;//cast to use implementation methods
        }
```

*5. Writing methods in the Binder class* 

```
public class MyService extends Service {

    private MyBinder mBinder = new MyBinder();
    private MyService myService = new MyService();
    
    public class MyBinder extends Binder{
        //Use MyService methods
        public void invokeServiceMethod(){
            myServiceMethod();
        }
        
        //Use MyBinder methods
        public void invokeMyBinderMethod(){
            
        }
        //Return service instance
        public MyService getMyService(){
            return myService;
        }
        
    }
    private void myServiceMethod(){
        
    }
    
    @Nullable
    @Override
    public IBinder onBind(Intent intent) {
        return mBinder;
    }

}
```

*6. Using the Binder object* 

```
    /**
     * Get myBinder/myService and call methods
     */
 @Override
        public void onServiceConnected(ComponentName componentName, IBinder iBinder) {
            MyService.MyBinder myBinder = (MyService.MyBinder) iBinder;
            myBinder.invokeServiceMethod();
            myBinder.invokeMyBinderMethod();
            MyService myService = myBinder.getMyService();
            myService.myServiceMethod();
        }
```
